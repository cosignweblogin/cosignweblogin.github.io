Note: When I set "AuthType Cosign" in my <directory> definition
(that was nested under a HTTP Basic required dir) then it did
not require that HTTP basic auth at all if you went directly
there without first visiting a higher directory, and if you
had already visited a higher directory and supplied your username,
the REMOTE_USER was always set as the Cosign username, so this
alleviated this problem.
